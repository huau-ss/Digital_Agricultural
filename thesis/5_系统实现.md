# 5 系统实现

## 5.1 开发环境

本系统的开发环境采用前后端分离的架构设计。

后端开发环境使用Python 3.11，Django 4.2框架配合Django REST Framework提供Web服务和API接口。数据库采用MySQL 8.0，通过mysqlclient驱动实现Python与数据库的连接。深度学习模块使用PyTorch 2.1实现LSTM神经网络的训练和预测，数据处理依赖pandas和numpy库。

前端开发环境使用Node.js作为运行时环境，Vue 3作为前端框架，Vite作为构建工具加快开发迭代速度。界面组件库采用Element Plus，图表展示使用ECharts。

开发工具选择上，后端推荐使用PyCharm进行Python开发，前端推荐使用VSCode配合Volar插件进行Vue开发。项目代码托管在Git仓库中，便于版本管理和团队协作。

## 5.2 数据采集与清洗模块实现

数据采集模块负责从外部数据源获取农产品价格信息。模块实现位于`backend/scripts/crawl_agri_prices.py`，核心类为`AgriPriceCrawler`。该类封装了数据抓取的主要逻辑，支持从农业农村部数据接口、批发市场数据接口等多个数据源采集数据。

采集流程首先建立与目标网站的HTTP连接，设置合理的请求头模拟浏览器访问。随后解析返回的页面内容，提取产品名称、类别、产地、价格、市场名称等关键字段。对于有效的数据记录，系统自动创建或更新农产品信息，并将价格数据存储到`PriceHistory`原始数据表中。

```python
class AgriPriceCrawler:
    def get_or_create_product(self, name: str, category: str, origin: str = '') -> AgriculturalProduct:
        """获取或创建农产品记录"""
        product = AgriculturalProduct.objects.filter(name=name).first()
        if not product:
            category_code = self.CATEGORY_MAPPING.get(category, 'other')
            product = AgriculturalProduct.objects.create(
                name=name, category=category_code, origin=origin, unit='元/公斤'
            )
        return product
```

数据清洗模块位于`backend/scripts/clean_price_data.py`，核心类为`PriceDataCleaner`。清洗流程包括删除重复记录、异常值检测和数据标准化三个主要步骤。

异常值检测采用IQR四分位距法和Z-score标准差法相结合的策略。对于每个产品的市场价格数据，计算其四分位数和标准差，将超出正常范围的数据标记为异常值。检测到的异常数据在`CleanedPriceData`表中标记`is_outlier=True`，后续的价格预测和收益分析只使用清洗后的正常数据。

```python
def detect_outliers_iqr(self, df: pd.DataFrame) -> pd.DataFrame:
    """使用IQR四分位距法检测异常值"""
    for (product_id, market_name), group in df.groupby(['product_id', 'market_name']):
        prices = pd.to_numeric(group['avg_price'], errors='coerce').dropna()
        Q1 = prices.quantile(0.25)
        Q3 = prices.quantile(0.75)
        IQR = Q3 - Q1
        lower_bound = Q1 - 1.5 * IQR
        upper_bound = Q3 + 1.5 * IQR
        mask = (prices < lower_bound) | (prices > upper_bound)
        # 超出范围的数据标记为异常值
```

## 5.3 LSTM价格预测模块实现

### 5.3.1 模型训练实现

LSTM模型训练脚本实现位于`backend/scripts/train_lstm.py`。训练流程从数据库读取清洗后的价格数据，经过归一化处理后使用滑动窗口技术构造训练样本。

数据预处理阶段使用MinMaxScaler将价格值归一化到0至1区间。滑动窗口长度设置为7天，即用连续7天的历史价格预测下一天的价格。归一化后的数据按时间顺序划分为训练集和测试集，比例为8比2。

模型结构采用双层LSTM设计，每层隐藏单元数为64。模型接收形状为(batch_size, 7, 1)的输入张量，经过两层LSTM层提取时序特征后，通过全连接层输出单一预测值。Dropout层设置丢弃率为0.2，有效防止模型过拟合。

```python
class PriceLSTM(nn.Module):
    def __init__(self, input_size=1, hidden_size=64, num_layers=2, output_size=1, dropout=0.2):
        super(PriceLSTM, self).__init__()
        self.lstm = nn.LSTM(
            input_size=input_size,
            hidden_size=hidden_size,
            num_layers=num_layers,
            batch_first=True,
            dropout=dropout if num_layers > 1 else 0
        )
        self.fc = nn.Linear(hidden_size, output_size)

    def forward(self, x):
        lstm_out, _ = self.lstm(x)
        output = lstm_out[:, -1, :]
        output = self.fc(output)
        return output
```

训练过程使用均方误差MSE作为损失函数，Adam优化器更新模型参数，学习率设置为0.001。训练过程中应用梯度裁剪技术，将梯度值限制在1.0以内以避免梯度爆炸。模型训练100轮后保存到`lstm_model.pth`文件，归一化器保存到`scaler.pkl`文件，训练过程中的损失变化绘制为曲线图保存。

### 5.3.2 价格预测功能实现

价格预测服务实现位于`backend/apps/data_analysis/lstm_service.py`，核心类为`LSTMPredictor`。该类封装了模型加载和数据预测的完整���辑。

预测流程首先加载已训练好的LSTM模型参数和归一化器。读取指定农产品的历史价格数据后，按滑动窗口格式组织输入数据。预测采用自回归策略，即用历史真实数据和已预测的值逐步推进，预测未来多天的价格走势。

```python
def predict_future(self, historical_prices, future_days=7):
    """自回归预测未来价格"""
    scaled_prices = self.scaler.transform(historical_prices.reshape(-1, 1))
    predictions = []
    current_seq = scaled_prices[-self.SEQ_LENGTH:].reshape(1, self.SEQ_LENGTH, 1)

    for _ in range(future_days):
        pred = self.model(torch.FloatTensor(current_seq))
        predictions.append(pred.item())
        # 更新输入序列进行下一步预测
        current_seq = np.roll(current_seq, -1, axis=1)
        current_seq[0, -1, 0] = pred.item()

    return self.scaler.inverse_transform(np.array(predictions).reshape(-1, 1))
```

预测结果返回历史价格和预测价格的时间序列，以及对应的日期列表。前端根据这些数据绘制价格对比图表，用不同颜色区分历史数据和预测数据，用虚线标识预测区间。

## 5.4 价格预警模块实现

价格预警模块实现位于`backend/apps/data_analysis/price_warning.py`。模块根据LSTM预测结果检测异常价格波动，并在触发预警条件时生成通知消息。

预警阈值设定为：预测价格涨幅超过15%或跌幅超过10%时触发预警通知。检测流程首先获取产品的当前市场价格，然后调用LSTM模型预测未来14天的价格走势。通过比较当前价格与预测的最高价和最低价，计算价格变化百分比。

当检测到异常波动时，系统根据用户角色生成差异化的预警消息。对于农户用户，价格上涨预警建议适当延迟销售以获得更高收益，价格下跌预警提醒及时调整销售策略。对于采购商用户，价格下跌预警建议推迟采购以降低成本，价格上涨预警提醒尽快完成采购避免成本增加。

预警消息存储到`SystemMessage`表中，包含消息标题、内容、优先级、关联产品等信息。用户登录后可查看预警通知，了解价格波动情况并做出相应决策。

## 5.5 收益模拟模块实现

收益模拟模块实现位于`frontend/src/views/analysis/ProfitSimulation.vue`。模块为不同角色用户提供收益分析和决策建议功能。

对于农户用户，模块收集种植面积、种子成本、化肥成本、农药成本、人工成本等种植参数，结合LSTM价格预测结果计算两种方案的收益对比。当前出售方案按当前市场价格计算预期利润，择期出售方案按预测最高价计算预期利润。系统生成个性化建议，包括建议推迟天数和预估额外收益。

```javascript
// 计算种植总成本和预期收益
const totalCost = form.seed_cost + form.fertilizer_cost +
                  form.pesticide_cost + form.labor_cost + form.other_cost
const totalYield = form.area * form.yield_per_acre
const currentProfit = currentPrice * totalYield - totalCost
const predictedProfit = predictedMaxPrice * totalYield - totalCost
const extraProfit = predictedProfit - currentProfit
```

对于采购商用户，模块收集采购数量、采购单价、物流成本、仓储成本、损耗率等参数，计算立即采购和择期待购两种方案的采购成本和收益差异。当预测价格呈下降趋势时，建议采购商推迟采购以降低成本；当预测价格上涨时，建议尽快完成采购锁定较低成本。

模块通过ECharts图表展示价格预测曲线和收益对比结果，用户可以直观地看到不同决策方案下的预期收益差异，从而做出更加科学的经营决策。

## 5.6 前端界面实现

### 5.6.1 数据可视化实现

前端数据可视化模块集成ECharts图表库，在仪表盘页面和数据大屏中展示丰富的可视化效果。

全国价格热力图使用中国地图组件，将不同省份的农产品平均价格用颜色深浅区分。价格较高的地区显示暖色调，价格较低的地区显示冷色调。用户可以直观地看到全国各省份的价格分布情况。

价格走势折线图展示单一产品历史价格的时间变化曲线，支持切换7天、15天、30天、60天等多种时间范围。图表支持鼠标悬停显示详细数据点信息，支持图表缩放和平移操作。

价格预测图将历史价格和LSTM预测价格绘制在同一坐标系中，用不同颜色和线型区分两类数据。历史数据使用实线，预测数据使用虚线，使图表清晰易读。

地区价格对比柱状图展示同一产品在不同市场的价格差异，按价格高低排序并用不同颜色着色，便于用户发现价格洼地和优化采购策略。

### 5.6.2 交互功能实现

前端交互功能基于Vue 3 Composition API实现。页面初始化时通过Axios调用后端API获取数据，数据返回后更新页面状态并渲染界面。

用户认证采用JWT令牌机制。登录成功后，服务器返回访问令牌，前端将令牌存储在本地存储中。后续请求通过Axios拦截器自动附加令牌，后端验证令牌有效性后返回请求数据。

```javascript
// Axios请求拦截器附加Token
axios.interceptors.request.use(config => {
    const token = localStorage.getItem('token')
    if (token) {
        config.headers.Authorization = `Bearer ${token}`
    }
    return config
})

// 获取仪表盘数据
const fetchDashboardData = async () => {
    const res = await getDashboardSummary()
    stats.value = res.data.stats || {}
    predictTop5.value = res.data.predict_top5 || []
}
```

表单交互使用Element Plus组件库构建。用户输入种植成本参数后，系统实时计算预期收益并更新界面显示。图表支持响应式布局，窗口大小变化时自动调整图表尺寸。

用户操作反馈通过ElMessage组件实现，包括成功提示、警告提示和错误提示。数据加载状态通过Loading组件遮罩显示，避免用户重复操作。
